# tabs

A minimal, mobile-friendly website for displaying song tabs (chords over
lyrics). Songs are authored in a plain-text format, stored in version control,
and statically generated into a site with [Astro](https://astro.build).

- **Static** output, near-zero JS (interactivity via small inline islands).
- **Catppuccin** theme — Mocha (dark, default) / Latte (light), Mauve accent.
- **Full-text search** with [Pagefind](https://pagefind.app) (title, artist,
  metadata, and lyrics).
- **Responsive chord rendering** — chords sit above their syllable and lines
  wrap; no monospace horizontal scroll.
- **Auto-scroll** with a play/pause button and speed slider, plus a combined
  "lyrics-only" toggle.

## Commands

This project uses [Bun](https://bun.sh). On NixOS, `nix develop` (or
`nix-shell -p bun`) puts `bun` on your PATH.

```sh
bun install        # install dependencies
bun run dev        # dev server at http://localhost:4321
bun test           # run the parser unit tests (vitest)
bun run build      # astro build + pagefind index into dist/
bun run preview    # serve the built static site
```

> Search needs the Pagefind index, which is generated by `bun run build`. In
> `dev` you must run a build first for search to return results.

## Song file format

Each song is a `*.txt` file in `src/content/songs/`. The filename (without
`.txt`) becomes the URL slug. A file has **YAML frontmatter** (the single source
of all metadata) followed by a **body** of sections.

```
---
title: Amazing Grace          # required
artist: Traditional           # required
album: English Ballad         # optional
key: G                        # optional
capo: 0                       # optional
tuning: Standard              # optional
year: 1779                    # optional
source: John Newton           # optional
tags:                         # optional
  - hymn
  - traditional
---

[Verse 1]
G                  G7        C        G
Amazing grace how sweet the sound
       G                          D
That saved a wretch like me
```

`title` and `artist` are **required** — a song missing either fails the build.

### Body rules

- A line of the form `[Name]` starts a new **section** (e.g. `[Verse 1]`,
  `[Chorus]`, `[Intro]`).
- Within a section, lines are read in pairs: a **chord line** directly above a
  **lyric line**. Each chord's horizontal column determines which syllable it
  belongs to, so you align chords by spacing them over the right letter.
- A non-blank line with no lyric line beneath it (followed by a blank line or
  end of section) renders as a **bare chord run** — handy for intros and
  instrumental breaks.
- Blank lines separate pairs and are otherwise ignored.

At build time the body is parsed (`src/lib/parseSong.ts`) into sections and
positioned chord/lyric segments, which render so chords stay above their
syllable while lines wrap on narrow screens.

## Project layout

```
src/
  content/
    config.ts              # 'songs' collection: custom loader + Zod schema
    songs/*.txt            # song files (frontmatter + custom body)
  lib/
    parseSong.ts           # body parser + chord/segment mapping
    parseSong.test.ts      # vitest unit tests
  layouts/BaseLayout.astro # head, no-flash theme script, header
  pages/
    index.astro            # search (Pagefind UI)
    songs/index.astro      # directory (filter + sort)
    songs/[slug].astro     # song page
  components/               # SongView, ChordLyricLine, ThemeToggle,
                            # ChordToggle, AutoScroll
  styles/                  # catppuccin.css, global.css
```
